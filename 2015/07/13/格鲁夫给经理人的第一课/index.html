格鲁夫给经理人的第一课

1. 早餐店的生产线

  1. 什么是生产：预定的时间、可接受的品质、最低的成本、依照客户需求，交付产品（或者服务）；
  2. 有哪些生产步骤：制造、组装、测试；（在软件行业，则是分析、设计、构建、集成、测试）
  3. 什么是限制步骤：整个生产过程中，需时最长（或最困难、最敏感、最昂贵）的步骤；(构建需要花的时间最久，所以优先考虑用敏捷开发的模式，小周期，快速迭代调整，验证获取信息，修正重新构建）
  4. 根据现有资源，如何找出最佳策略：了解生产过程中的每一个步骤及其互动关系，了解各方案的利弊得失；（调研和设计先行，构建使用敏捷，测试集成到开发中）
  5. 测试与验货；及早发现，及早解决；发现的越早，解决的成本越低；（测试要提早到开发的每一个单元中，越早进行越好）
  6. 如何设定衡量指标，确保业务有在正常运转？我们关心的事情有哪些？从此处入手；（快速迭代，找用户验证）
  7. 关于指标的两个原则：有总比没有好（衡量结果，不衡量过程）；衡量具体且可计算的事情；（估算的准确率、速率）（可用性测试，测试通过的指标是什么）
  8. 指标配对：避免过度反应，在不同取向的指标中取得平衡，找到最佳方案；
  9. 指标的3个好处：1、清楚列出了目标；2、为评价考核提供客观性；3、为不同组织相同工作提供比较的可能性；
  10. 黑箱理论：将生产活动视为黑箱=投入+产出+人力；
  11. 几种指标：先行指标（了解未来、引导现在）、线性指标（例如燃尽图）、趋势指标（同比、环比）、重复印证表；
  12. 品质保证：关卡式、监视器式、随机式；
  13. 产能=全部产出/投入人力；提高产能的3个办法：1、加速生产；2、改变工作本质（该做什么）；3、提高杠杆率；
  14. 提高杠杆率办法：自动化、简化工作步骤；

2. 打好团体战

  1. 经理人的产出=他直接管辖部门的产出+他间接影响所及部门的产出；
  2. 书面报告的作用：用来对自己的问题或方案进行更严格的审视，即它的自律性大于信息传递作用；
  3. 一个极有效的收集信息的办法：多在公司内走动；
  4. 经理人的产出计算公式=活动AA的杠杆率+活动BB的杠杆率……以此类推，所以最重要的一件事情是，找出杠杆率最高的2-3项活动，一心一意去做；
  5. 授权的必要条件：授权人与被授权人有相同的信息基础，且二者对开展工作和解决问题有彼此认同的方法；
  6. 没有完备的监督计划的授权等于渎职；监督不是干涉，而是不时的检查来确定活动的进行一如预期；
  7. 监督的节奏，在工作成熟度最低的时候，在活动早期及时进行监督；当成熟度上升，减少监督的频率；
  8. 提高工作速度的方法：
    1. 找出限制步骤（列出非做不可的事）；
    2. 类似的工作集中在一起做；
    3. 安排好日程表，在非做不可的事情中，安排穿插一些”很重要但时间有弹性的事”来做；
    4. 学会拒绝超过工作负荷的事；
    5. 建立指标，预估事项的完成时间；
    6. 储备存货：一些不急着完成的项目；
    7. 标准化；
  9. 部属人数：如果“带人”是主要职责，那么6~8个，确保每个部属可以分配到足够的时间；（那如果不是以带人为主要职责呢？）
  10. 应对干扰和突发情况：1、将不规律的事情规律化，即制定处理手册的东西；2、划分固定的时间来处理，其他时间不接单；
  11. 开会：过程导向会（知识技能和信息的交流）；任务导向会（为解决某个问题）；前者通过规律化提高会议效率，比如一对一会议（互通信息以及彼此学习）；
  12. 1对1会议：频率（新手一周一次，熟手一月一次）；时间（长到足够提出棘手问题）；部署主持（准备会议纲要）；要点：由部署提出潜在问题，上司扮演协调者的角色，让部署畅言；技巧：上司要写下来部署的问题；同时建立存档，将一些重要不紧急的事项列下来留待下次讨论，分批处理；
  13. 部门会议：涉及2人以上的工作事项，在部门会议中讨论；上司可以通过此会议，比1对1会议更加了解事情的真相本质；上司在会议中扮演协调者的角色，确保不跑题、控制进度、化解纷争；
  14. 运营总结会议：让由于组织关系，没有参加1对1和部门会议的人员之间，可以有机会彼此学习和分享经验；会议由提案人上司来召集并指导提案人讲演节奏；由总结负责人来带动自由讨论；由提案人在讲解；由其他人发表意见和提出疑问；（作用：讨论提案，学习经验）
  15. 任务导向会议：
    1. 会议主席在会前必须弄清楚哪些决策需要在会上进行制定，通过开会想得到什么，有没有其他不开会的办法，如果有则不要开；
    2. 决定开会后，确定出席人并确保参加，如果不能参加，请他找个能代表他说话的人来；
    3. 与会人员不超过6~7个，不然有点推不动；
    4. 确保每个参会人在会前拿到议程，知道会议目的以及每个人要扮演的角色；
    5. 会后通过发送会议记录确定决策的内容；会议记录越快发出越好，并尽可能详细，让看的人知道有什么事该做、谁负责去做以及什么时候做；
  16. 决策：对于以信息和科技为主导的公司，决策应该让离问题越近，最了解问题的人来制定；（那非信息和科技主导的公司呢？比如哪些公司？手工制造业？服务业？农业？我觉得可能是那种主要以执行为主的行业，不可预测性较少，不需要前线的人员随机应变；比如说钢铁厂、粮食加工行业，大型原材料制作业）
  17. 理想决策过程的三阶段：
    1. 自由讨论；
    2. 清楚的决策；
    3. 全力支持；
    4. 务必注意一点：决策内容的制定和执行应该交给最低层级；
  18. 注意同级群体综合证，注意加一个更高职位的人，来引导讨论不偏题；
  19. 制定决策前应了然的6个问题：
    1. 决策内容
    2. 决策时限：如果不限定时限，有可能会导致问题被无限期拖延；因为很多人害怕承担责任；
    3. 决策人
    4. 制定决策前应向谁咨询
    5. 谁是决策终结者：决策终结者可以考虑在决策过程中不参与决定制定，只是倾听和提出疑问，最后的时刻，才出来发表意见；
    6. 谁应该在决策后被告知；
  20. 规划：做规划的目的相当于一种未雨绸缪，而不是等问题来了才临时应变；
    1. 预测需求（界定外部因素，现在与未来）；（设想未来的场景，提前为这个场景的到来做准备）
    2. 分析现状（定义目前的状况）；
    3. 缩小差距（决定要采取哪些办法，需要做些什么，自己能做什么）；
  21. 规划的产出：一套行动方案（注意，不是定出目标而已，这个需要非常明确）
    1. 什么事情今天做了，明天会更好，或至少不变坏；
    2. 规划可能达5年，但实际的行动方案是接下来的1年，因为等下一步再做规划的时候，会修改后续的行动方案；
    3. 规划的人应该是负责执行的人；
  22. 目标管理：（目标通常是短期的）（因为太长期的目标由于诸多不可预测性，可能需要经常变更，导致失去意义）
    1. 我想去哪里？（答案即目标）；
    2. 我如何知道正朝着目标迈进？（沿途应验收的成果；验收应依据经理人的才智经验来判断；）

3. 推动组织的巧手

  1. 任务导向型组织（优点：能快速响应市场需求；缺点：成本高）；（或许可以通过阿米巴方式来解决）
  2. 功能导向型组织（优点：能发挥规模效应，成本低；缺点：跨部门协调导致决策响应慢）；
  3. 混血型组织（优缺点折中）；
  4. 双重报告：既满足快速响应需求的需要，又能发挥规模效应；
  5. 人的行为受３种因素影响：自由市场因素；契约关系；文化价值观；（同一个人在不同阶段中的主导因素可能不同）；

4. 谋事在人

  1. 激励部属参加比赛；马斯洛的需求模型；创造讲究产出的环境；制定游戏规则，让员工有衡量表现的尺度；求胜但不怕输，并随时向自己的极限挑战；（如果想象不出来游戏规则，或许可以组织大家头脑风暴，来共同设计游戏规划）
  2. 管理风格取决于工作成熟度：
    1. 低：告诉该做什么、如何着手、什么时候完成；
    2. 中：沟通、情绪上的支持与鼓励为主；
    3. 高：参与程度低，建立起工作目标及监督系统；
    4. 无论哪个阶段，都应随时且适度的检查部属的工作，以避免出现突发状况；
  3. 绩效评估：
    1. 检视部属的技能水准（看看哪些技能缺乏有待提高）；
    2. 加强激励的力度（让其创造出更高的绩效）；
  4. 绩效评估考虑：
    1. 产出评估（应对现在）与流程评估（应付未来）；
    2. 长期绩效和短期绩效（将长期绩效折现）；
    3. 产出与时间的关系；
    4. 个人与部门兼顾（评估经理人适用）；
    5. 避免潜力陷阱（只评估绩效而非潜力）；
  5. 告知评估结果：坦诚、倾听、忘了自己；只谈最重要的4点（多了吸收不了）；提前告知评估结果，之后双方再1对1讨论；花更多时间在评估优秀的员工上面（他们更值得投入更多的时间）；
  6. 招人：
    1. 80%的时间让对方讲自己关心的话题；
    2. 收集信息：专业知识如何、应用知识的能力；
    3. 询问假设性的问题；
    4. 让面试者提问；
    5. 与推荐人谈一谈；
  7. 留人：如果一名极具价值的员工提出辞职，以第一时间推掉所有事情，并倾听他的想法；给予尊重和肯定，即使需要将其推荐到其他部门；
  8. 报酬要明显的反映绩效；
    1. 对金钱敏感的人员（如中低层员工），绩效奖金在报酬中的比例应为偏小；
    2. 对金钱不敏感的人，绩效奖金在报酬中的比例应越大；
    3. 分清楚考核的是个人绩效还是团队绩效；
    4. 注意绩效奖金发放的时间尽量接近工作完成之时（以便员工记得为何受到奖励）；
  9. 除非我们能在部属之间分出高下，根据表现或是折中的薪资制度才能够运作；（如果我们分不出高下，那么我们无法根据他们的表现给予客观评估，这样会让人感觉不公平）
  10. 升迁：考虑多个维度，管理维度和技术维度；如果升迁后无法胜任，则不要怕再回收；
  11. 绩效考核：公平、公正、公开；
  12. 培训：经理人应该扛起培训的责任；因为：部门产出即经理人的产出；
  13. 培训的持续性：
    1. 即有系统、有计划；一直进行，而非单一事件；
    2. 培训应与公司实务相结合；
    3. 培训由足以成为员工楷模的人员来担任（讲课的人需要有可信度与权威性）；
  14. 两类培训：
    1. 新进员工需要掌握的知识和技能；
    2. 现有员工需要掌握的新知识和新技能；
  15. 培训课程的设置：
    1. 可以问问员工他们需要学一些什么；
    2. 优先级：先为最紧要的项目设计短期培训课程（3-4堂课）；
    3. 第一堂课通常不太成功，可以先给有经验的人试讲，收集反馈改进课程设计；
    4. 如果课程需要覆盖的人很多，可以考虑培训出几个讲师代劳；
    5. 上完课后请用上课的学员提出匿名批评；

5. 期末考 询问自己以下的问题

  1. 试着将你工作中的操作分为编制流程、组装及测试三个步骤；（我觉得这里说的操作是指操作人员的实务操作，不是经理人员的操作；这样就比较好理解了，比如调研的流程、设计的流程、测试的流程等，这些工作都可以设计工作流程；）
    1. 调研的流程
      1. 确定调研的目的；
      2. 选择调研的方法；
      3. 调研前的工具、资料、人员准备；
      4. 执行调研，收集信息；
      5. 分析数据，得到调研结论，出具调研报告；
    2. 设计的流程
      1. 战略层：确认目标
        1. 公司的目的；
        2. 用户的目的；
      2. 范围层：确认要做什么，不做什么，先做什么；
      3. 结构层：如何安排交互，让用户完成他的任务目标；
      4. 框架层：如何排版布局，设计界面，让用户一目了然；
      5. 表现层：如何加强用户的视觉感知，辅助其完成任务；
    3. 测试的流程
      1. 测试的原型、人员、资料准备；
      2. 执行测试，收集信息；
      3. 分析信息，得出结论和建议；
  2. 针对你手头开展的方案，找出限制步骤，并依此设计你的工作流程；（限制步骤一般指那个耗时最长、或成本最高的步骤，一般工作流程要围绕它进行优化，包括找了依赖关系，安排并行事项等）
  3. 找出你工作中最适合进行验货、线上检验与最终检验的地方；决定这些检验应该采用关卡还是监视器的方式；然后考虑在什么时机下，你可以升格至“随机检验”；（限制步骤开始前，最好确认一下前期工作是否都准备好，以及相关人员对限制步骤的开展是否具备知识和技能；最后再安排检查结果）（至于是关卡还是监视器或随机检验，取决于成员的工作成熟度；如果成熟度低，则每个流程步骤都需要进行验收，甚至在每个步骤前告知应做什么，怎么样，何时完成；）
  4. 找出至少6项的新产出指标；这些指标应该要能衡量产出的质与量；（质与量的目的是什么？是为了最后的按时按质按需求最低成本交付，所以指标包括：完成时间、完成质量、需求匹配程度、成本；如果再细分，每一项需要怎么分？完成时间=预算耗时+实际耗时；质量=数据完整+逻辑严谨+条理清晰+结论有力；需求匹配=用户净推荐率+相对价值法（让用户列出工作中感觉好用的软件，打基准分；之后再给本软件打相对分；或许可考虑让用户匿名打分投票）；成本=消耗的时间+投入的人员+其他直接费用；）（指标更多只是为了观察工作是否顺利进行，不列入绩效考核，考核因为以交付客户价值来衡量比较准确有力）
  5. 将这些新的指标变成工作上的例行事项，并在部门会议中定期审视；
  6. 你现在正寻找的最重要的战略（行动计划）是什么？描述你面临的环境需求以及计划进度。如果计划能成功执行，是否能将你或你的公司带到理想中的境界？（我理想中的境界是什么？提出有价值的方案，获得客户的订单，带来收入，创建利润；所以我的计划应该包括寻找有价值的解决方案、向客户展示价值、按时按质依需求最低成本交付）
  7. 简化你最烦琐、最耗时的工作。至少让原有的步骤减少30%；
  8. 找出什么是你真正的产出？你所管理的部门及影响力所及的部门产出元素为何？按重要顺序排列；
  9. 实际在公司中走动走动；然后，列出这次“出巡”中和你有关的事项；
  10. 找一些借口让你一个月可以在公司内巡视一次；
  11. 描述下一次你授权给部属时会如何督导；你将以什么为标准；怎么做；督导的频率是怎样的；
  12. 列出你可以利用零碎的时间进行的项目；
  13. 列出和每一个部属1对1会议的时间表（在会议之前向他们解释会议的目的，并要求他们做准备）；（会议的目的是由其反馈问题，以便发现情况，及协调指导）
  14. 找出你上星期的日程表，将做的活动分为高、中、低杠杆率三类；设法多做一些高杠杆率的活动；（有哪些活动该减少或干脆不做；）
  15. 预测下周有哪些事要瓜分你的时间；有多少时间要花在开会上？其中有多少是过程导向会议？多少是任务导向会议？如果后者占据的时间超过25%，你该如何设法删减？（任务导向会议应如何删减？）
  16. 列出你的组织在未来三个月中最重要的三个目标，并一路验收成果；
  17. 在与部属充分讨论过以上目标后，要他们也“依葫芦画瓢”-制定他们的目标并一路验收；
  18. 写出“悬而未决”需作决策的事项；找出其中三项，试着运用决策制定过程的架构以及“六点问题”的方法；（决策内容、决策时限、决策人、谁应该在决策前被咨询、谁是决策终结者、谁应该在决策后被告知）
  19. 依据马斯洛的需求理论评估你自己的需求层次；并为部属找出他们所属的层次；
  20. 给部属勾画他们的跑道，并找出每一个人的绩效指标；
  21. 列出你给部属各种形式的工作所给予的相关回馈；他们是否能借着这些回馈来测量自己的进度；
  22. 将你部属的工作成熟度分低、中、高三类；并针对个人选出最适当的管理风格并在你的管理风格及最适当的管理风格之间作比较；
  23. 评估你上一次收到的或你对部属所作的绩效报告，这些报告对提高绩效有多大的影响；在上司告诉你报告内容或你告诉部属时，你们的沟通形式是怎样的；
  24. 如果有哪一份报告不够理想，重做；
