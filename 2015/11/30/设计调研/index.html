设计调研

第一章 数据采集
1.1 观察法

  1. 几个属性：
    1. 布景：人工布景模拟真实环境，或者直接在真实环境中观察；
    2. 结构：
      1. 开放式：没有限制和采取任何提前措施，直接观察并记录；
      2. 结构式：先预观察，然后分类，之后大量观察前，对用户进行分组培训，再观察统计；
    3. 公开性：用户是否知道自己在被观察；不知道最好，但有时需要用户配合，不可避免需要让用户知道；
    4. 参与水平：观察人是否介入事件中；
  2. 步骤：
    1. 明确观察方向：观察谁，观察什么事，在什么环境下观察；
    2. 观察的准备：选择观察对象，观察地点（实地，或者实验室）；
    3. 观察框架：该观察什么，怎么记录，后期如何整理分析；
      1. POEMS： PEOPLE, OBJECTS, ENVIRONMENT, MESSAGE, SERVICES;
      2. 预观察：1~2个预观察可以提高后续大批量观察的效率；
      3. 进行观察：看、听、问、思、记；
    4. 整理分析：观察后越快整理越好，因为记忆更加鲜活；
  3. 优缺点：信息量大，数据真实性高；成本高，耗时长；
    1.2 单人访谈法
  4. 通过访谈可以了解到用户的思考过程，这是观察法所无法做到的；另外还可以了解用户的观点和看法；
  5. 访谈结构：介绍（访谈的目的）、暖场（让对方放松）、一般问题、深入问题、回顾与总结（段落间）、结束语与感谢；
  6. 访谈问题类型（一般是两种结合使用）
    1. 结构化问题（询问结果可以直接作为选择题目的答案）；
    2. 开放式问题（无固定的回答格式）；
  7. 访谈技巧：
    1. 主持：随和、不主观偏见；
    2. 提问方式：开放式，不引导；
    3. 倾听与回应；
    4. 重复与释义；
    5. 跟进与深挖；
  8. 访谈环境：注意双方的座位安排形成谈话气氛，环境不大不小，目的是放对方感到放松；
  9. 步骤：
    1. 确定目标被访者：明确研究的目的，提出基本的问题，之后整理筛选条件；
    2. 挑选目标群体；
    3. 安排日程和发出邀请；
    4. 做好充分的事前准备：仔细检查每一个细节，包括时间、材料、设备等；
    5. 常见问题及解决办法：找不到人（发动关系网）；找错人（多找一定比例的额外人员）；失约（让邀请显得正式）；迟到（预留一定的机动时间）；隐私保护（做好资料存档）；
      1.3 焦点小组
  10. 特点：某一个主题，结构化的方式，揭示用户的经验、感受、态度、愿望和理由；
  11. 优点：由于人多，之间可以互相启发，有利于发现用户的愿望、动机、态度和理由；不同用户的观点可以碰撞，获得直观对比；
  12. 缺点：多用于探索需求、态度为主，结论多以描述用户的看法和感受为主；故主要用于定性，而较少用于定量；群体之间会相互影响；
  13. 步骤：介绍（座谈的目的）、访谈规则（表达真实想法、无对错、围绕一定的问题开展、依次回答如有补充需等待他人发言完毕、控制过程和时间）
  14. 座谈的不同目的和类型：
    1. 挖掘需求型：开放式问题，让用户在轻松的氛围中，描述自己的生活场景，诉说自己的喜好和痛点，表达自己的想象和建议；
    2. 验证结论型：收集用户相互碰撞思想和对话之后的建议；
    3. 竞品对比型：比较结构化的访谈大纲，以便可以针对竞品之间的不同点做出比较选择，获得相关评价和讨论；
  15. 访谈大纲：
    1. 通过确认好的主题确定板块结构：比如一般问题+深入问题；
    2. 撰写合适的问题；
    3. 分配问题的访谈时间；
    4. 准备材料、工具、设备；
    5. 如果需要，设计试前问卷和试中问卷；
  16. 访谈技巧：平均分配、鼓励引导；提供思考时间，不要主导；
  17. 访谈记录办法：观察、助理研究员、做笔录、影音、允许用户追加补充问题；
  18. 数据整理：材料和数据的分类、准备数据表格、补充遗漏；
  19. 常见问题：
    1. 意见领袖（控制他的影响，注意数据质量）；
    2. 隐藏想法（鼓励表达、优先发问）；
    3. 争论（对事不对人，不分对错，适时打断情绪激动者）；
    4. 数据遗漏（边记录边核查原有的问题清单并提醒主持人）；

1.4 问卷法

  1. 分类：
    1. 结构问卷（问题具体/回答快/易统计，但限制多）；非结构问卷（限制少/内容丰富，但难分析）；半结构问卷；
    2. 代填问卷（街访/电话时使用）；自填问卷（邮寄/网络/报刊）；
    3. 问卷结构：介绍语、指导语、问题+答案、编码（方便后续统计分析）；
    4. 问题设计：
      1. 开放式问题：问题+空白；
      2. 封闭式问题：问题+答案；
      3. 后续性问题：类似选择式展开的模式；
    5. 问题设计注意事项：
      1. 必须非常非常明确研究的目的；
      2. 问题应当具体，不能抽象笼统；
      3. 避免出现复合型问题，此情况建议拆成两个问题；
      4. 问题通俗易懂，用尽量简单的语言，少出现专业术语；
      5. 避免倾向性和诱导性问题；
      6. 不用否定式提问；
      7. 不提敏感性或威胁性问题；
      8. 敏感问题注意适度模糊、转移对象、采用假定法、提供背景信息或辅助信息；
    6. 题目数量：尽量在20分钟内可以答完；如果以结构方式询问，并给予一定报酬或礼品，回收质量会比较高；
    7. 问题顺序：先易后难，先简后繁，先事实后主观；同类主题的内容放在一起，方便思考连续性；逻辑验证问题要隔开，避免干扰；
    8. 答案设计：是非型、选项型、排序型、等级型、模拟线性型、视觉模拟型；
    9. 问卷调查实施：
      1. 人数选择按一定比例多于需要的研究对象；（因为回收率的问题）
      2. 前期摸底，有利于提高设计的质量；
      3. 设计初稿可采用卡片法和框图法；
      4. 通过初稿的试用反馈来改进问卷设计质量；
    10. 优点：不受空间限制、方便分析、可减少偏见和误差、匿名性、省时省力省钱、短期内可收集大量反馈；
    11. 缺点：获得的信息有限、调查样本影响质量、回收率可能不高、难以了解用户的意图和动机以及思维过程（观察法也一样）；
      1.5 头脑风暴法
  2. 以收集创意为目的，召集一批有知识素养和不同专业背景的成员，进行集体讨论，相互启发和激励；
  3. 常见方法：
    1. 默写式：653模式，6个人，5分钟，3个想法；每人发3张纸，5分钟内在每张纸上各写1个想法；之后传给下一人，进行下一个5分钟；（大家可以有时间静静思考，避免争着发言导致设想遗漏）；
    2. 卡片法：每人在自己的卡片上写下设想，之后向大家介绍；大家在倾听后进行讨论，之后在原有卡片上进行设想补充（此方法更侧重对想法进行论证，完善可行性）
    3. 电子法：让大家可以同时在电子板上面录入，避免了卡片一次性只能一个人在填写的物理局限性；（感觉像群聊）
    4. 专家函询法：找20名左右的专家，发函询收集设想，之后进行整理设想，隐去姓名；再发第二轮函询；一般发4-5轮；可以减少权威意见的影响力，弱化表达能力的限制，避免随大流；
  4. 操作流程：
    1. 会前准备：确定议题和任务目标；找5-10名参与人员；准备场所/设备（录音摄像）/记录人；必要的话，进行会前训练（用其他有趣的议题模拟演习一下，针对出现的问题，提前引导大家采用正确的方式）；
    2. 实施阶段：议题通报，回答疑问；第1轮的轮流发言，记录整理；第2轮的轮流发言，记录整理；中场休息，调节气氛，如唱歌，吃零食等；在预定的结束时间，给大家1分钟补充想法，如果没有新设想，则宣告结束；
    3. 成果整理：可以采用思维导图的工具；会中整理/会后整理/成果报告；
  5. 原则：
    1. 自由发散：营造轻松氛围，鼓励天马行空；
    2. 主题聚焦：如果有跑题，记得及时拉回；
    3. 以量求质：多设想，多思考，暂不考虑实现成本；
    4. 延时评判：不对任何设想做任何评价；
  6. 优点：自由气氛及小组讨论带来的共振效应激发更多的设想产生；由于不考虑成本，可以尽量排除折中方案；
  7. 缺点：产生式阻碍；评价焦虑；社会惰化；认识干扰；
    1.6 自我陈述法
  8. 用户对自己的使用过程和使用经历的回顾和描述；比较适用于产品上线后的反馈收集；
  9. 特点：不干预、用户主动反馈、在使用场景中进行思考；一般是主动性较高的用户；
  10. 原则：让用户在轻松的氛围中完成反馈，形式不限，在线反馈亦可，语音反馈亦可；
  11. 反馈表的三个层次：
    1. 对整体印象的评价；
    2. 对主要功能进行排序，标识出最有帮助或最易操作的功能；
    3. 在既定的业务场景中的专业问题反馈；
  12. 优点：维持了用户的自然使用状态；用户可以在较长时间内自主使用产品；反馈都在积极主动的前提下反生，数据较真实可信；
  13. 缺点：由于形式不限导致内容可能分析困难；反馈时间不好控制；时间和内容的不确定性导致较难用于系统性的调研；
    1.7 现场试验法（感觉A/B测试法也可以归类到这里）
  14. 优点：由于控制了自变量，可以看出变量之间的因果关系；
  15. 缺点：现场背景较难把握，成本及花费可能较高；
  16. 定量试验的步骤：
    1. 拟定并提出研究假设；
    2. 选定试验对象；
    3. 决定需要操纵的试验变量及确定所要观察的变数；
    4. 设法排除无关的变量；
    5. 使用恒定的测量工具和人员进行测度；
    6. 选择适当的统计方法进行分析；
  17. 定性试验的步骤：
    1. 准备工作：选择测试者进入现场的身份/方式/途径；列好观察与访谈的提纲；预告安装摄像头或麦克风；
    2. 取样：挑选合适的受试者；
    3. 收集数据：最好安排3名测试者，一名专门负责记录，一名负责深度访谈，一名负责观察受试者；
    4. 数据处理/分析/解释：可以通过POEMS框架对受试者的交互行为做标签（POEMS：人/物/环境/消息/服务）；
      第二章 调研分析
  18. 数量对比分析
    1. 数量之间的关系包括：大小关系、占比关系、趋势变化、相关性；
    2. 表现方式：
      1. 表格：由行、列、单元格组成，可以丰富的容纳数据，并通过排序/筛选/数据透视表等方法进行统计分析；
      2. 图表：以图形化的方式来进行文字描述，直观的表示数据的一种形式；
        1. 饼图/环状图：主要用来表示占比关系；
        2. 柱状图：常用来表示时序关系；
          1. 直方图：常用来统计次数，如不同年龄段的数量；
          2. 分组长条图：常用于分组内数值的比较；
          3. 瀑布图：几个特定数值间的关系；
        3. 条形图：
          1. 横条图；
          2. 排序长条图；
        4. 折线图；
          1. 折线图：常用于处理连续性数据的变化关系；
          2. 点折线图：数据较少，需要强调数据点，以及趋势变化关系；
          3. 多重折线图：有利于各数据系列进行整体比较；
        5. 散点图：两个变量之间的关联关系；
        6. 雷达图：头尾相边的折线图；（可以从多个维度来对比各自的优缺点）
        7. 面积图：折线下方填充颜色，比多重折线图看起来更加直观；
        8. 气泡图：除了横纵坐标两个维度外，还可以通过气泡面积增加第三个数据维度；
      3. 数量大小对比分析：
        1. 同一个数据在不同时间点的比较；
        2. 不同数据在相同时间点的比较；
        3. 不同数据在同一个领域内的比较；
      4. 趋势变化对比分析：上升趋势、下降趋势、波动趋势、保持水平不变的趋势；
      5. 占比关系对比分析；
      6. 相关性的对比分析：正比相关、反比相关、制约相关（某个决定性因素存在时vs不存在时）；
  19. 知觉图&鱼骨图
    1. 知觉图：不知道怎么绘制出来的专业玩意，忽略它；
    2. 鱼骨图：大骨（现象）-小骨（现象的原因）-小小骨（原因的原因）；
      1. 五个为什么，追根究底找出深层次的原因；
      2. 详细思考一一罗列想到的原因，避免遗漏；
      3. 步骤：确认问题，讨论绘制，确认优先级/严重程度/解决方案；
      4. 优点：直观/有条理的呈现事情的原因
      5. 缺点：需要有了解问题的专家参与；不能呈现问题的优先级；
  20. 卡片法
    1. 用于对信息进行分类，适用的场合有：导航设计、信息架构设计等；通过分类过程和结果，可体验到目标用户群体迥然不同的心智模型；
    2. 优点：可视化、可移动、可沟通；缺点：小样本可能代表性不足、小组讨论可能受成员表达能力的限制；
    3. 操作步骤：
      1. 准备工作：
        1. 卡片：大小合适，既可以写得下内容，又不太大方便单手操作；
        2. 数量约30左右；多准备几张空白的；
        3. 颜色：如果有不同颜色可以方便进行分类；
        4. 场所：方便走动；
        5. 参与者：一般6-10人左右比较合适；
        6. 纸/笔/小礼物；
      2. 试验过程：
        1. 卡片下面写内容（文字建议经过讨论使其尽量容易理解）、背面写序号以方便统计；
        2. 介绍规则；
        3. 参与者进行分类：可单独进行，也可分小组进行；
        4. 研究者进行观察，如果有疑问可进行解答，必要的时候可以修改卡片文字以促进理解；
        5. 分类结束后，回顾自己的分类过程，如果有不清晰的地方可以重新思考一下；
        6. 确定分类后，可以让参与者讲一下分类逻辑；
        7. 感谢，发礼物；
        8. 注意事项：
          1. 尽量让真实用户来参与，相对研究者自己参与，会更有启发性；
          2. 不需要参与者分类严谨而合乎逻辑，而鼓励他们跟随自己的感觉走；（我们是为了了解他们的心智模型）
      3. 线性Delphi法：虽然比较耗时，但是更具启发性；
        1. 先由用户各自独立进行分类；
        2. 然后各人介绍、解释自己的分类；他人可以提出疑问，但不允许批评；
        3. 之后各人吸取他人的想法后，回去调整自己的方案；
        4. 之后再次聚集起来分享各自的想法，并解释自己与众不同的想法与理由；
        5. 之后再回去修改，周而复始，直到最后大家的想法趋于一致；
      4. 案例：儿童需求研究与儿童房设计
        1. 确定目标用户群体，通过访谈了解目标用户一天的行为；
        2. 收集数据后，将每一个行为记录到卡片上，进行第1轮关于用户行为的分类；
        3. 分类结束后，提炼每一种行为背后的生理和心理需求，再将需求写在新的卡片上，进行第2轮关于用户需求分类；
        4. 分类结束后，运用角色法将需求归纳为6类家庭模型；
  21. 情景分析法
    1. 通过故事的形式生动形象的迅速描绘用户执行任务时的大致情况；目的是为了让相关人员能够对用户的形成统一的认识，了解用户的行为习惯，弄清楚用户的行为，便于产品的进一步改进；
    2. 几个应用的场合：
      1. 对用户行为的梳理：与观察法、访谈法配合使用；
      2. 做为头脑风暴的素材；
      3. 用于验证产品方案是否符合用户的习惯；
      4. 与角色法配合使用，使得用户的行为路径更加清晰；
    3. 操作步骤：
      1. 设想归纳一个大概的故事主线；
      2. 收集故事的元素：
        1. 一个特定的环境与状态；
        2. 一个或多个角色（描述他们的动机、能力和知识）；
        3. 和角色互动的工具或者物体；
        4. 注意事项：
          1. 这个场景发生的一系列事件和最后的结果；
          2. 用户为达成目标所制定的计划，以及对结果产生的反应；
      3. 整理完善用户故事的撰写（某人在某个条件下做了什么为什么最后结果是什么）；
      4. 标注情景故事中的要点，包括：
        1. 主要行为流程图；
        2. 标注故事中的物品以及状态；
        3. 标注机会点；
    4. 优点：方便大家对用户行为形成统一认识；缺点：虽然形象，但如果操作不当，可能描述并不准确而导致方向搞错；
  22. 人物角色法
    1. 基于真实人物的目标/行为/观点/动机，综合成一组对典型产品使用者的描述；作用：辅助设计，帮忙设计师跳出“为自己设计”的惯性思维，立足于用户的角度看待问题；时刻提醒关键需求/关键任务/关键流程；促进团队成员间的沟通和对目标用户的理解；
    2. 操作步骤：
      1. 发现用户：谁是目标用户？有多少？
      2. 建立假设：思考用户之间的差异是什么；
      3. 调研：访谈、观察、问卷、可用性测试；内容：喜欢/不喜欢、内在需求、价值倾向、工作环境、工作策略和目标、信息策略和目标；
      4. 发现共同模式：是否有重要的标签？是否同等重要？是否有更多的用户群；
      5. 构造虚拟角色：基本信息、背景、对待技术的态度、个人特质等；
      6. 定义场景：需求在什么样的场景下发生；
      7. 创建场景：在既定人物/既定条件下使用产品后，会发生什么？
      8. 持续的改进：根据新收集的数据，不断完善角色；
    3. 人物模型卡：
      1. 基本档案：年龄/性别/收入/职业/教育背景/照片等（生动合适的照片很重要）；
      2. 人格：大五类人格特征（高大上的玩意，暂时不知道怎么整）；
      3. 会发生影响的对象：比如用户使用的网络/设备/软件等；
      4. 原型分类和引语（一句能够代表其特质的话）；
      5. 技能：IT方面的技能；
      6. 用户体验目标：让用户感觉易用/安全/有趣等；
      7. 用户曾经使用过的设备和平台（影响其学习上手速度和可能存在的先入为主的观念）；
      8. 使用习惯和兴趣：经常使用的软件有哪些，使用时间有多少，可以用饼图表示比例关系；
      9. 哪些必须做/哪些不可以做：可以制定一些交互和体验方面和规则；
      10. 与品牌和产品的关系：新用户/老用户/粉丝用户等；
    4. 注意：通常设计只能满足1-2个主要角色模型，如果要满足太多角色，可能是反思是否产品定位太模糊宽泛了；
    5. 优点：用户越是精确分群，越可以帮忙产品定位清晰，功能差异化，从而实现商业的成功；使得设计人员聚焦主要角色，重点关注用户的目标/动机/行为/观点，使得研发人员有统一的视角；
  23. 故事板
    1. 通过图画的方式，讲好角色/情景/故事，优点是更容易有代入感和沉浸感，发现原文字描述所容易忽视的细节；
    2. 四个要素：人/物/场景/事件；
    3. 步骤：确认角色、构建场景、讲一个故事、画图；
  24. 可用性测试
    1. 通过原型来测试产品的可用性；
    2. 步骤：
      1. 整理功能点；
      2. 根据功能点创建任务（任务是根据用户的心理过程或心理目标来设计的，所以，在可用性测试之前需要通过观察法和访谈法等方式来搞清楚用户的心理目标）；
    3. 大纲和脚本：
      1. 大纲是主试人员在做测试时的流程规范，内容包括：任务和功能、记录点、备注等；
      2. 脚本是主试人员在测试过程中要讲的台词、要提问的问题、要做的事的一个汇总；
      3. 记录纸：记录纸一般由观察员来记，如果没有人手，可以通过事后观看录像来记录，内容主要为客观记录观察到的事项；
    4. 数据整理：
      1. 收集结果：定性部分（主要描述性的文字内容如用户的操作思路/操作理由等）、定量部分（次数、耗时、频率、费用等）；
      2. 建立问题卡片：相当于建立JIRA任务，内容可包括：问题类型、问题位置、问题描述、重要程度、问题建议、后续措施等；
      3. 问题评级：以下是几个维度
        1. 对用户继续使用的影响程度是大还是小；
        2. 对用户的感受情绪是重还是轻；
        3. 问题在整个产品中是主要问题还是次要问题；
        4. 问题影响的时间是长还是短；
      4. 正常通过8-10名测试人员，可以发现70%-85%的问题；
  25. A/B测试
    1. 可以通过专门的工具来部署，只要提前设定好不同版本的方案即可，可包括同一个页面上的不同元素，或者设置不同页面的URL；
    2. 步骤：
      1. 提出问题：想解决什么事情；
      2. 提出假设：这个事情的产生可能是由于什么原因引起的；
      3. 提出方案：根据假设，设计不同的方案；
      4. 部署A/B测试：通过专门的工具部署不同的方案；
      5. 分析结果报告：验证假设是否正确；
      6. 新一轮的循环；
    3. 测试的内容：标题、图片、视频、段落文字、按钮、表格、颜色、社会认同和信任、确认页面；
    4. 优点：快速、简易、成本低、有专门工具、对比作用明显、变量设置灵活；
    5. 缺点：适用可量化的指标，不适用不可量化的指标；只适用短期效果，不方便对比长期效果；无法洞察用户更具体的心理细节；
    6. 工具：
      1. Google Website Optimizer;
      2. A/Bingo and Vanity;
      3. Visual Website Optimizer;
      4. Unbounce and Performable;
      5. Verster, SiteSpect, Webtrends Optimizer and Omniture’s Test&Target;（企业级测试工具）
  26. 用户点击行为分析
    1. 用户点击界面产生的数据，经过分析和挖掘后，得到一些有用的帮助，包括：
      1. 判断页面的内容和结构是否符合产品设计的预期（即页面设计了一些功能，但用户操作的时候，是否能够按设计的设想来完成）；
      2. 判断用户对页面上的哪些内容感兴趣；
      3. 帮助到达该页面的用户更容易找到或者发现他们更容易关注的内容，提高用户的满意度；
    2. 相对于眼动跟踪，这种方法更容易实施，成本也比较低，同时数据也不失客观有效；
    3. 相对于网页跳转跟踪法，这种方式更倾向于分析用户在单个页面内的微观行为；
    4. 实现办法：JS代码，第三方工具等；
    5. 实际案例的收获（腾讯应用中心）：
      1. 清晰简洁的页面框架更有利于用户观察到页面内容本身，从而让他们更容易注意到和点击他们感兴趣的内容；
      2. 整齐而简洁的信息单元轮廓（即格式塔的闭合性），有利于提高用户对该模块的点击可能性；（研究表明，低视觉复杂程度的网页，能够让用户感觉到更愉悦，并带来更高的满意度）
      3. 某个模块如果需要展示的内容较多，但又不想让它占用较多的纵向空间，则可以考虑使用动态轮播；
      4. 在通过点击数据分析了解到用户更关注哪些内容后，就可以更多的展示相同类别的内容；（量身推荐效果好的原因：相比让用户主动查阅新东西，他们更喜欢去点击他们已有的或生活中已熟悉的事物相关的一些内容）
  27. 流量、转化率和跳出率
    1. 网站分析：根据网站运营的各项定量指标，分析网站存在的问题和机会；
    2. 操作步骤：根据需要的指标预告在网站上设置监控（“埋点”），之后运作一段时间后取得数据；
    3. 理念：数据驱动业务！！！（试问：我们做到了没有？）
    4. 流量：PV访问次数；UV访问人数（估计是按IP统计）；
    5. 转化率：用户到达网站设计的最终目标的比例；
      1. 统计办法：网站流量统计的第三方工具、网站日志文件；
      2. 提高转化率的办法：
        1. 优化网站信息架构和导航，让用户能够方便的找到想要的内容；（往前）
        2. 介绍的信息丰富、专业，让同时停留更长时间，同时增加信任感；（当前）
        3. 操作流程简单，让用户可以尽快达成浏览网站的目的；（往后）
        4. 合同定价和多种推广、营销手段，促使用户快速转换（例如：达成交易）；（大背景）
    6. 跳出率：用户通过搜索关键词进入网站，只浏览了一个页面就离开与全部浏览量的比例；
      1. 反映对网站不感兴趣的用户的比例，是判断网站质量的重要指标；
      2. 降低办法：
        1. 导航尽量合理，方便用户访问其他感兴趣的内容；
        2. 链接正确友好，避免用户有被欺骗的感觉；
        3. 提高网站访问速度：快速、快速、快速，重要的事情说三遍；
        4. 网页内容符合用户需要；
        5. 做好关联推荐，方便用户了解更多感兴趣的信息；
  28. 网站数据分析
    1. 一些重要的数据指标：
      1. 用户是如何到达网站的
        1. 直接流量/推介流量/搜索流量；
        2. 访问数（PV/UV）；
      2. 用户是如何浏览网站的
        1. 平均访问时长；
        2. 每次访问页数；
        3. 页面停留时间；
        4. 网站停留时间
      3. 用户在网站的关键行为
        1. 目标转化次数；
        2. 目标转化率；
        3. 每次转化成本；
      4. 用户对网站的收益贡献
        1. 订单数量；
        2. 每订单成本；
        3. 收益；
        4. 投资回报率；
      5. 有多少用户流失：跳出率；
    2. 数据分析手段：
      1. 网站日志分析；
      2. 网页标记法；

第三章 从设计调研到设计洞察

  1. 设计调研的流程
    1. 只有先确定调研好目标，才好选择合适的调研方法；
    2. 调研对应产品不同阶段的大致过程：了解情况、形成方案、选择方案、收集反馈；
    3. 制定调研计划的步骤：
      1. 弄清楚调研的背景及目的；
      2. 出具调研方案：包括确定调研方法、安排调研顺序、选择调研对象、安排调研时间表；
      3. 出具调研报告：（定性）包括目的、结论、证据、建议；（定量）建议采用图表的方式，结果比纯数字更加直观；报告提交前，建议先找几个人演示解说，根据反馈意见改进后，再最终定稿；
    4. 执行调研过程
      1. 产品探索阶段：多采用观察法、访谈法、焦点小组、情境法等；
      2. 产品方案阶段：可用性测试、卡片分类法；
