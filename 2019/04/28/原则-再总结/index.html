原则-再总结

1. 生活原则

  1. 拥抱和应对现实：自学、试错、请教、反思
    1. 自学：理解现实的运作规律，目的：唯有掌握规律，才能有效应对；
    2. 试错：通过快速试错去理解现实价值无限；
    3. 请教：在自己不擅长的领域善于请教他人；原因：没有人能够客观看待自己，时刻记得依赖他人的意见和证据；
    4. 痛思：痛苦+反思=进步；原因：绝大多数痛苦来源于应对现实的能力不足；
  2. 用五步法实现目标
    1. 目标：明确且有优先顺序的目标；
    2. 问题：找到阻碍实现目标的问题；
    3. 根源：剖析问题的根本原因；
    4. 方案：规划问题的解决方案；
    5. 实践：实践问题的解决方案；
  3. 做到头脑极度开放：谦逊、理解、寻找、逻辑
    1. 谦逊：认识到个人思维障碍的客观存在，包括自我防御和思维盲点，保持谦逊之心；
    2. 理解：先尝试理解并提问，而不是先尝试说服；
    3. 寻找：理解他人，利用他人的帮助，找出最佳方案，而不是靠自己想出；
    4. 逻辑：重视证据+逻辑；
  4. 理解人与人大不同：有同有不同，人人两个我，取长补短，互相协作；
    1. 不同：每个人思维方式不同；避免浪费时间在改进沟通方式上面，而是先尝试去理解对方，之后按照对方的逻辑进行推理，并不断补充和完善；
    2. 相同：有意义的工作和人际关系，是人的天性追求；
    3. 双我：控制大脑中的低层次和高层次的两个自我；
    4. 优缺：借助心理学工具，客观评估每个人的擅长与不擅长的领域；
    5. 协作：团队成员各有所长，各司其职，是成功的关键；
  5. 学会有效决策： 第一步先收集信息，完成后，再进入第二步决策；
    1. 信息：决定问谁，区分观点，抓大放小
      1. 问谁：可信度高的人；
      2. 区分：观点和事实；观点很廉价，事实则只有一种；
      3. 大小：抓大放小，不过分沉溺于不重要的细节信息；
    2. 决策：排除情绪，求助集体，逻辑分层，概率原则
      1. 分层：将决策拆分成要点，将其他细项归纳在要点下面，形成层次，避免被低层次的细节浪费了时间；
      2. 原则：引用原则，将每次决策用到的原则写下来，为后续决策提供参考；以及后续根据决策结果回头完善原则；
      3. 逻辑：证据+逻辑+理性+常识
      4. 概率：决策依据 = 成功收益成功概率 > 失败损失失败概率；
      5. 情绪：情绪是影响决策的最大有害阻碍因素，应该排除；
      6. 集体：求助他人协助决策，以投票按可信度加权；讨论分歧前，先就原则达成一致；

2. 工作原则

1. 文化（价值观）：透明容错、关系共识、创意择优

  1. 求真透明（原因：了解真相是解决问题达成目标的基础；）
    1. 如果公开不会带来损失，则应尽量公开；例外事项尽可能少；
    2. 善于保密者更加透明，不擅长者应不透明，甚至请其离开
    3. 知情者负责管理信息的义务；
    4. 表达异议是一种义务；有不同意见不可不说；
    5. 不相信不诚实的人；让惯于撒谎者离开；
  2. 允许犯错：
    1. 发现问题时，有义务将问题转交给擅长处理问题的高手，以最大程度减少损失；
    2. 要从错误中吸取教训，制定措施避免下次出现相同问题；
    3. 不可重复犯错
  3. 追求关系：追求有意义的人际关系；
    1. 思考：什么样的关系让人感觉有意义？为什么家人的关系让我们感觉有意义？
    2. 对于父母：被需要、被依赖、被信任、被尊重；（员工对公司）
    3. 对于子女：被爱、被支持；（上级对下级）
    4. 体贴宽厚：用体贴宽厚来促进合作的情感，逆境时才能渡过难关；
  4. 求取共识
    1. 原因：一次性投入，长久产出；
    2. 方式：抱怨低层开放清单，方式辩证的理解案例，
      1. 罗列清单：把可能的分歧，按优先级罗列成清单，为下一步讨论做准备；
      2. 区分抱怨：区分并排除狭隘的无意义抱怨，识别有意义的改进诉求；
      3. 控制低层：不要将建议识别为批评，不要让低层次的情绪影响自己高层次的理性
      4. 开放心态：负责人必须有以开放心态应对异议的能力；提防那些不敢承认自己无知的人；
      5. 确认理解：双方都有责任确保自己先复核并理解对方的观点后，才做下一步的讨论；
      6. 方式另论：若不认同对方的沟通方式，可以先写下来，待处理完当前议题后，作为下一项议题讨论；
      7. 辩证思考：凡事皆有得失，应看到事物的两面性，而不是绝对的倾向某一方面；
      8. 引用案例：以案例来说明抽象的原则如何应用；
  5. 创意择优
    1. 擅长的领域有更高的权重；擅长的标准有两条：
      1. 曾经成功解决过类似的问题
      2. 能清楚的解释逻辑和因果关系；

2. 人：做什么事情不重要，最重要的是找到合适的人

  1. 找人：流程价值不将就、各有德才能进化；
    1. 科学流程：而非凭感觉，最重要的事情包括：挑选、培训、测试、评估、调配；
    2. 不可将就：找出色的人，而不是某类即可；
    3. 价值一致：彼此愿意分享人生的人；
    4. 各有所长：避免只有同类人；各有所长，相互配合才更有战斗力；
    5. 德才兼备：核查情况，过去只能用来了解对方是什么样的人（德）；能否胜任，有逻辑的工作思路更重要（才）
    6. 自省成长：会问好问题，愿意客观认识自己的缺点，并持续不断进化；
  2. 用人：两分配、两匹配；
    1. 分配上级：确保每个人都有上级；
    2. 分配工作：定目标，找对人，聊思路，给培训，给时间，评估结果；
    3. 责权匹配：承担责任的同时，确保权利相配，不然没有资源覆行责任；
    4. 职人匹配：明确岗位所需素质，人员需与岗位匹配；
  3. 管人：培训、测试、评估、调配
    1. 价值观：实现成长
      1. 每个人都需要不断经历成长，不完美不可怕，可怕的是持续保持现状，拒绝成长；
      2. 目标是为了实现快速成长；不完美是允许的，就像允许犯错一样，重点是最快的时间找出不完美的地方，有针对性的改进，实现快速成长；成长的速度，比是否已经完美更值得追求，因为前者是动态的，后者是静态的；
    2. 管理者职责：培训、反馈、评估、调配
    3. 流程设计
      1. 培训：制定清晰规则、原则、案例、指标，通过培训让员工学习；
        1. 原因：规则可减少争议、指标用于形成记录，为客观评估提供参考；
      2. 反馈：宏观、持续、数据
        1. 持续性：原因：避免进入评估时让人感觉突兀；
        2. 宏观性；协助员工宏观思考；原因：以更宏观的视角看待自己，更容易明确方向，取得更多的进步；
        3. 数据化：注重客观，过程数据化；原因：数据可避免偏见和情绪，目的是让员工认清缺点，有针对性的改进，实现快速成长；
      3. 评估：
        1. 方法：基于案例，谦逊辩证；控制情绪，展示同理心，给出建议；
          1. 案例考核；原因：基于证据，方便达成共识；
          2. 辩证评价：原因：有得即有失，看清二者，避免偏颇；目标是找出规律，把握全局；
          3. 保持谦逊：每个人都只能掌握部分事实，应避免自信，要求基于证据，让对方一起补充证据；原因：保持评估过程客观理性，避免陷入情绪；
          4. 控制情绪：听到批评，人会感到难受是正常的，这是成长之痛；若情绪失控，可先暂停谈话，等过几天再继续；
          5. 同理心+建议：批评的同时，要展示同理心，人的优点与盲点是并存的；同时给出建议，这样才能带来鼓舞，化消极为积极；
        2. 原则：抽查、二八、姿势
          1. 使用抽查：不必事无巨细，重点是了解对方是什么样的人；区别对待事前审批和事后审计；原因：通过保持抽查，可避免由于人的天性导致的松懈；
          2. 挥杆姿势：考察做事的思维方式和逻辑过程，比做了什么事更重要（多看挥杆姿势，少看是否命中）；
            1. 务必区分能力不足，还是经验不够；后者容易弥补，前者则很难；
            2. 陷阱：只评估是否掌握技能，未评估是否具备能力，后者更重要；
            3. 单个事例的成败缺少说服力，要看长期多次的统计；
          3. 不求完美：应使用二八法则，确保大方向无误即可，不要在小细节上过于纠缠；在重点的20%的投入获得80%的回报即可（就像一次养成一个重要的好习惯就够了）
      4. 调配：按能、不修、不兼；
        1. 不要修复：改变员工的价值观和能力是很难的，如果这二项不合格，不要尝试修复，而应辞退；
        2. 鱼和熊掌：我们无法得到一切，为了公司长远牺牲情谊有时候是必要的，不然会动摇文化的根基；原因：留下不合适员工的破坏力惊人；辞退人是项艰难的决定，但却是必要的；管理者肩负公司所有人的信任，容忍的代价不是只损害管理者的利益，而是损害整个公司其他所有人的利益；
        3. 按能调岗：如果有能力，但不适应现在的岗位，则可以调配到可以发挥能力的岗位上；适应岗位需要时间，避免调整的过快和过于频繁，给予时间磨合；
  4. 人员薪酬：合理空间，因人而异，区分慷慨，
    1. 追求合理，不必追求精确；
    2. 区分慷慨大方和公平合理；
    3. 依人发薪：永远不要依头衔发薪；
    4. 增长空间：基本薪资提高财务稳定性，余下空间提供奋斗动机；总体薪酬高于一般水平；
  5. 团队规模：保持简小：1 + 1 = 3 和 3~5 > 20
    1. 原因：人数太多的团队，超过某个临界点会边际效益递减
    2. 贝索斯：两个披萨喂不饱的团队就太大了
      1. 原因：阿米巴原则，让公司成为制造机器的机器；每个阿米巴要有自己与外界的沟通机制；如果没有，则应该有一个阿米巴去创建并满足这个需求；
      2. 后来，我又想到，应该让原则成为一种基因；由于机器变大不可避免会带来效率下降，因此更好的做法是效法自然，同一套基因，生产多个个体，每个个体有自己的大脑，这样个体可以对自然界做出自己的反应；兼顾效率与灵活性；在旧的成熟领域，大是一种优势；在新的新兴领域，小才是优势，因此有必要让新旧两个领域的团队，各自拥有不同的大脑；
      3. 新大脑什么时候脱离旧大脑？答：原则的内化需要时间，就像婴儿到成人需要时间一样；当原则已经内化成为新组织的习惯行为时，即是它独立的那一天；
    3. 原因分析：
      1. 团队大小跟工作内容的性质有很大关系；随着团队规模的扩大，成员之间的信息沟通线路呈指数级上升，复杂度可能会失控；如果一种工作，要求成员之间充分的沟通，则应保持小规模；如果成员之间基本不需要沟通，则可以大规模，例如流水线的工人；
      2. 为维持有意义的关系， 人们需要付出一定的时间成本，当团队规模过大，人均维护成本过低，关系的质量会变淡；导致失去了有意义的人际关系；
  6. 团队组成：共同价值观；如果团队成员的价值观不同，则应该重新组合；原因：团队成员应该基于相同的价值观，才能发挥最大的产出，否则会花很多时间在处理冲突上面；

3. 机器

  1. 建造机器：双层、宏观、职责；挑战、利益、问责、备选；每日、上报、审计；
    1. 宏观视角：
      1. 迭代式设计：了解机器工作原理，建造机器，然后持续监控、调试和改进，确保给出输入后，它能产生预期的结果
        1. 就像写一个函数一样，或许整个公司就像一个大函数嵌套一些小函数；
      2. 数字字管理：通过数字指标，确保它正常运行；
        1. 要了解机器是否正常运转，需要问出哪些问题？
        2. 使用哪些数字，可以为这些问题提供答案？
      3. 绕开陷阱：时刻明确自己的职责是建造机器，避免被微观事务分散注意力，而忘了站在宏观层面进行机器设计的思考和实践；
    2. 双层思考
      1. 接近目标：解决当前问题
      2. 改进机器：让每个问题都形成案例，剖析原因，探究根源，制订原则，改进机器，转为沉淀，用于培训；
    3. 责任到人：目标到人、责任清晰；目的：避免角色错位，责任混乱；按照流程设计和人员能力，分配工作职责，而不是按头衔；例如业务部门负责录用决策，而不是人力部门；
    4. 利益绑定：设计让每人的决策关系其自身利益的机制，以便把钱当作自己的钱来花；
    5. 挑战目标：目标的制定要有一定的挑战性，能够让人振奋；原因：一定强度以上的锻炼才能达到保持强壮的效果；
    6. 关键备选：识别关键员工，承认风险，安排继任人选，加以训练以便能够履行职责；
    7. 建立问责：越级、相互、透明
      1. 相互问责：上下级之间相互问责，列入日常工作；目的：避免问题发生的太突然；促使进步，因为没人可以客观的看待自己；
      2. 越级问责：触及下属的下级，了解下属如何管理下级；允许下下级越级汇报问题；
      3. 透明问责：让问责过程透明，而非私下问责；目的：保证问责的质量，强化极度透明的文化；
    8. 每日更新：第一时间了解情况，并对情况进行交叉验证，目的：全面了解，知谁可信；促进准确性；人的天性是做喜欢和容易的事，而不是重要却困难的事；通过每日更新的跟踪，来确保方案按正确方式在开展；
    9. 上报问题：没有能力完成目标不是问题，但有责任第一时间主动上报问题，由主管安排其他人处理，或者给出处理方案；
    10. 建立审计：人的本性是自私，不要在人性上下赌注；让审计成为例行机制，原因：突然袭击会让员工觉得对他们不信任；审计人员也需要被审计；当发生违规行为时，杀鸡儆猴，同时让所有人知道后果；不要容忍欺骗，因为它是一种习惯，不要奢望改善
  2. 维护机器：
    1. 人：了解、关心、理解，让步、下沉；了解、提问、约定、寻找、逻辑；
      1. 了解员工：将与员工的谈话列入日常工作；目的：发现问题、建立授权信心、获得正确预期；
      2. 关心员工：如果不与自己关心和尊重的人一起共事，工作就没有深层的意义
      3. 让人理解：而不是通过发号施令来让他人顺从；
      4. 轻勿让步：重要的是做正确的事，而不是别人如何看我们，让步会释放原则可以随意更改的信号；
      5. 避免下沉：作为管理者，职责是建造和管理机器，而不是代替员工做他们应该做的具体事务；对问题进行分层，避免在低层次的细节上浪费自己的时间；
      6. 了解领域：足够了解所管理的领域，避免被借口和故事糊弄；
      7. 善于提问：向有智慧的人寻求建议，让比自己强的人来指挥；
      8. 书面约定：失败有两种情况，一种是没有遵守约定，一种是根本没有约定；不要想当然，应书面记录约定，确保提前达成共识；
      9. 寻找答案：领导的职责在于以开明的态度寻找答案，并带领他人一起寻找；当旧方案需要花费很多时间时，退一步，停下来，寻找有创意、新颖的解决方案；
      10. 关注逻辑：重点关注决策是否符合逻辑，立场客观；而不是别人是否喜欢或赞同，人的本性是希望别人相信其观点，通过转变职责为寻找答案来克服这个本性；
    2. 流程：想象和执行计划、利用杠杆助手和检修清单，外包风暴；
      1. 决策清单：将原则列成清单，决策后对照清单进行；原因：避免感性、情绪和遗漏；目的：将最重要的原则，变成一种行为习惯；
      2. 想象计划：想象自己在电影中走完流程，体验和发现潜在问题，寻找不同的选项，考虑计划给当下和未来可能带来的后果；定期召开标准化的会议，让计划像钟表一样进行；
      3. 执行计划：
        1. 计划执行前：员工应理解本部门目标和任务计划，给出反馈，建立信心；
        2. 计划执行中：量化指标评估进度；
        3. 计划要变更：改前先沟通征求意见，目的：评估进展与尽职；推新计划先回顾旧计划；
      4. 清洗风暴：阶段性的“清洗”风暴，目的：提高系统的反脆弱性；
        1. 思考：自然界中的风暴，会刮倒脆弱的小树，留下强壮的大树，在资源有限的情况下，让大树得到更多资源和成长空间；因此，对于公司而言，相同的是资源有限，清洗的目标是淘汰弱小，让资源更集中到强壮的对象上面；因此，清洗的对象应该是以部门为单位，而不是要清洗人；人是公司的资源，他们不是清洗的对象；不具备成长性的业务部门才是要清洗的对象；
      5. 追求杠杆：让一份工作时间的投入，取得多份的产生，例如沉淀知识做成培训课程、寻求自动化重复工作的办法，都是好例子；向能力强的人，请教高效做事的新方法；
      6. 善用助手：概念性思维只需10%的时间，剩下90%的执行事项，应交由优秀的助手来实施；
      7. 慎用外包：必要的时候可以使用，但不能变成依赖；原因：外包可以解决短期问题，但同时会失去长期效益，包括成本过高、道德风险、无法沉淀；
      8. 定时检修：不断的工作会消耗过大，让人厌倦，应定时停下来，检讨过去，调整自己和机器，以便能够重新出发；（这点有些像敏捷中的冲刺结束阶段，或许可以尝试三周冲刺，一周检修）
  3. 发现问题，不容忍问题：设计能够自动暴露问题并自行改进的机器；
    1. 价值观：危机感、不容忍、不旁观
      1. 危机感：人无远虑，必有近忧；
      2. 不容忍：人对环境有适应性，不容忍小问题，不然就会对各种问题习以为常；
      3. 不旁观：每个人都有责任汇报问题，不能因为可能会汇报，就减少自己的责任；
    2. 方法：专人、讨论、发言、个人、解决、进展；
      1. 专人负责：指定专人负责发现和汇报问题，给予时间审查，独立汇报路线，不需承担后果；
      2. 鼓励讨论：鼓励多人共同讨论问题，利用不同角度的思维，探究问题的根源；
      3. 前线发言：最熟悉工作的人，最有发言权；
      4. 原因到人：分析问题的原因，要具体到个人，而不是使用集体名义来掩盖问题根源；允许犯错，鼓励试错；但不允许掩盖错误，而应第一时间汇报错误和问题；
      5. 马上解决：人的天性是逃避困难的问题，但它会恶化局面；所以发现问题后应马上解决，方法包括提交上一级，转移给问题处理高手等；
      6. 量化进展：设计指标，量化问题的解决进展，应为日常工作的一部分；
  4. 剖析问题，探究根源；只有找到根源，才能真正的解决问题
    1. 剖析原则
      1. 不马后炮：挑出别人的毛病很容易，但不代表自己能做得更好；在当时信息不明的情况下，具备能力的人，会去了解什么和做什么？
      2. 不看环境：环境优劣不可控，但应对方法优劣可控；重点是评估操作方法是否合理；
    2. 剖析目标：找有无规律：问题的根源原因是否有规律性？
    3. 剖析办法：找责任人，找根本原因
      1. 找责任人：结果是好的，还是不好的？谁是这个结果的负责人？
      2. 找根本原因：五个为什么；远离、能力、经验；敏感、自负、害怕；不懂、原则；机器、缺人
        1. 能力不够：相同能力的人，是否也会出现相同的错误？
        2. 经验不足：是否缺少相关工作情景的培训或处理原则？
        3. 违反原则：是否有哪些原则被违反了？
        4. 人手不足：假设人手充足，是否能够保证成功？
        5. 过于自负：不愿意承认每个人都有盲点，不愿请教他人，寻找最佳答案？
        6. 远离问题：不清楚前线发生的情况？
        7. 害怕失败：害怕承担失败的后果？
        8. 不懂质量：无法辨别质量的高低？
        9. 无敏感度：对问题习以为常，没有敏感度？
        10. 机器异常：机器是否在正常运转？如果 X 因素下次不出现，是否能避免发生不好的结果？
  5. 解决问题
    1. 制定计划：直接、参与、平衡、具体
      1. 直接关联：计划应直指问题根源本身；
      2. 共同参与：责任人和关键员工共同参与制订计划，并就计划达成共识；
      3. 平衡具体：平衡风险、成本、收益、可行性；计划要有具体责任人、具体任务事项、时间表、进度跟踪和结果反馈指标等；
    2. 执行计划：责任、公开、汇报、调整
      1. 负起责任：让员工对计划的成功和及时完成负有责任；
      2. 公开透明：各方同意后以公开透明的方式进行跟踪；
      3. 定时汇报：至少按月报告执行的进展（计划的进度、实际的进度、未来的进度）；
      4. 按需调整：若现实情况有变，根据需要调整计划；
  6. 组织架构：上下合并目标，交叉问责继任，规模比例自足，做两遍
    1. 围绕目标：而非围绕任务；若目标不同，即使任务相同，也应该分成两个团队，避免造成混乱；
    2. 自给自足：尽量让各部门控制实现目标的所需资源，原因：避免浪费时间在对资源池的争夺上；
    3. 避免交叉：避免混乱；避免职责错位，例如支持部门误以为自己可以干预决策；不帮其他部门解决问题，也不借调其他部门人员，除非对方同意；争议交由交叉点处理；
    4. 不乱合并：尤其是当部门工作性质相差很大时；原因：管理者需要深入了解下属工作，具备管理下属的能力，按大类乱合并部门会导致出现无效管理；
    5. 控制规模：原因：规模上升导致效率下降，在考虑扩大规模，优先考虑利用各种提高效率的工具；
    6. 控制比例：正常 1：5 左右，不超过 1：10，原因：多了无法有效沟通；
    7. 自上而下：先招管理者，再招下属；原因：管理者是机器建造者，可以帮忙设计岗位；
    8. 安排问责：每个人都需要分配一名高标准、高可信度的问责人；原因：不要在人性的弱点上下赌注；（此点类似于前面的“分配上级”？）
    9. 继任计划：即使某个人不在，机器也能正常运转；接班前，一起思考，一起决策，在实践中锻炼成长；（沉淀原则、进行培训，授权实践）
    10. 做两遍：关键问题应该使用做两遍的方法，而不是使用二次确认；原因：确保两遍之间独立不相关，这样才能暴露问题，避免流为形式；
  7. 正确开会：个人理解互补，目标原则一致，逻辑抓大放小，专人两分钟
    1. 个人目标：开会以实现个人目标为导向，该个体应为会议负责人；原因：避免方向跑偏；
    2. 精简互补：参加人员少而精，尽量由不同思维方式的人组成；
    3. 理解问题：确保与会的每个人，理解会议负责人想要解决的问题；
    4. 原则一致：先就原则层面达成一致，再讨论具体的问题；（或可考虑先将原则写在黑板上或者其他一目了然的地方）
    5. 逻辑事实：坚持让观点基于逻辑与事实进行，而不凭感觉；
    6. 抓大放小：评估细节的潜在收益，不高则放弃；高则择日再议；
    7. 两分钟法：避免有人说太多，占用其他人的发言时间；
    8. 专人负责：专人记录、专人承诺、专人跟进、专人汇报；

4. 工具：辅助、实践、公信、择优

  1. 目的
    1. 落地实践：仅学会知识并不会带来改变，重点是实践知识，设计工具帮助人们实践知识；制订原则，培训原则后，设计工具让原则有效落地；
    2. 辅助决策：收集数据，设计算法，辅助决策；
    3. 营造公信：使用工具，让任何结论都可以跟踪其背后的逻辑和数据加以评估，形成公平和信任的氛围；
    4. 创意择优：让每个人都可以参与原则、决策、评估标准的制定，然后通过创意择优机制选用，减少争议；
  2. 简介：案例、集点、棒球；痛苦、分歧、问题；每日、契约；流程、政策、指标
    1. 案例库：
      1. 用法：记录具体案例、当时决策所引用的原则；以及后续执行过程中，对原则的改进和完善；
      2. 目的：查询引用、改进完善；
      3. 思考
        1. 哪些决策需要进入案例库？答：案例库中没有的新问题；
    2. 集点器：
      1. 用法：预先设置多个维度，给每个人的观点打分；
      2. 目的：
        1. 宏观视角，让人保持开放与谦卑；
        2. 收集反馈，创意择优；
    3. 棒球卡：
      1. 用法：预先设计透明且让人理解的算法，对每个人的阶段成果进行评估，以便确定每个人的长短项；
      2. 目的：
        1. 为可信度提供数据支撑；
        2. 寻找与岗位的匹配人选；
    4. 问题日志
      1. 用法：当出现问题时，提高到日志中，问题处理能手跟进，记录处理方案，以及避免问题再次发生的系统性的改善方案
      2. 目的：
        1. 统计处理机制，避免散乱；
        2. 持续评估改进处理方案；
    5. 痛苦按钮
      1. 用法：将工作中感受到的消极情绪，用日志记录下来，默认私密状态，也可选择公开；
      2. 目的：
        1. 为回顾和反思提供参考，因为痛苦+反思=进步；
        2. 为心理咨询医生提供参考；
    6. 分歧解决器
      1. 用法：当两个人之间存在不同意见时，先写到分歧解决器中，之后会引入可信度高的人，帮忙评估是否需要提高给更高层级的管理人员进行研究处理
      2. 目的：
        1. 让每个人的意见都有发声的渠道，避免被埋没；
    7. 每日更新
      1. 用法：每人每天花10-15分钟的时间，记录当天所处理的工作，遇到的问题，反思与收获等；
      2. 目的：
        1. 跟踪工作进展；
        2. 第一时间了解问题的发生；
        3. 了解员工遇到的困难；判断员工工作负荷是否过大；
    8. 契约记录
      1. 用法：将人们在开会时的口头承诺记录下来
      2. 目的：协助追踪当事人某项承诺的履行情况；
    9. 作业流程图
      1. 用法：为公司每个部门的日常例行事项设计流程，电子化，同时关联每个环节的负责人；
      2. 目的：
        1. 方便随时随地查阅，知悉对机器进行输入后，将会如何得到最终的结果
        2. 了解每个环节的岗位要求，与人员能力的匹配情况；（通过关联棒球卡完成）
    10. 政策和程序手册
      1. 用法：各种政策和程序的汇编，动态更新，支持在线查询
      2. 目的：公司对经验积累的书面化；可用于学习和培训；
    11. 量化指标
      1. 用法：设计指标，跟踪指标
      2. 目的：用机器来辅助管理，减少大脑的工作量
      3. 设计指标的四个步骤
        1. 了解目标是什么；
        2. 了解完成目标的程序：人和流程设计；
        3. 找到程序中最适合量化的部分，以便通过这个部分，了解机器的运作情况；
        4. 研究如何在关键指标上发挥杠杆作用，以最小的投入，得到最大的产出；
      4. 判断指标有效性的标准：是否可以通过指标，了解到具体某个人在某个领域的表现好坏，一定要具体到个人；
