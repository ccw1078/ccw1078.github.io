重新定义团队

  1. 成为一名创始人
    1. 要有创始人的心态，不一定真的要去创建一家公司，即使在一个小团队内也可以；
    2. 把自己看做是一名创造人，并像创始人一样行动；
  2. 文化可以将战略像早餐一样吃掉
    1. 一项有意义的使命；
      1. 宽泛一些，不要局限于客户，因为客户会变，而且有少数客户也不一定是好人；
      2. 在内心深处，每个人都想要找到工作的意义；
      3. 让员工与他们帮助的人会面，是一项很有效的激发意义的办法；
    2. 如果相信员工，就不必害怕跟他们分享信息；
      1. 保持透明；
    3. 我们所有人都想掌握自己的命运；
      1. 允许每一个人发声；毫无保留的说出他们的观点；
    4. 打造了不起的文化
      1. 将工作看做是一种命运的召唤，而且工作要有富于意义的使命；
      2. 给人员工稍多于你舒适区的信任、自由和自主权；如果你没有感到紧张，可能是因为你给的还不够多；
  3. 关于招聘
    1. 在资源有限的情况下，把人力资源的费用首先投入到招聘上；
    2. 慢慢来，只聘用最优秀的人；聘用那些在某些特定方面比你更优秀的人才
    3. 不要让经理独立做团队人员聘用的决策；
  4. 搜寻最优人才
    1. 详细说明寻找人才的标准，并根据这个标准找到最优秀的被推荐人；
    2. 使招聘成为每个人的工作，让员工帮忙推荐优秀的人才；
    3. 可以考虑尝试一些疯狂的主意，以便引起一些优秀人才的注意力；
  5. 不要相信你的直觉
    1. 预测某人未来的工作表现的几种较有效的方法：
      1. 样例测试（29%）：一些在实际工作中发生的问题
      2. 一般认知能力测试（26%）：
      3. 结构化测试（26%）：行为测试（过去的某些行为）和情境测试（某些虚拟情境下的处理方式）
      4. 尽责性（10%）
      5. 领导力；
    2. 网上可以找到一些结构化测试样题，稍加修改，即可以成为面试题目；
      1. https://www.va.gov/pbi/questions.asp
    3. 面试反馈意见包括：对某个特性的评估，面试问题，应聘者的答案，面试官对答案的评估；
      1. 特性有：一般认知能力、领导力、尽责性、职务相关知识；
      2. 每一项特性至少有两名面试官独立做出评估；
      3. 每名面试官不一定需要评估所有特性；
    4. 如何做到：
      1. 设定高质量的招聘标准；
      2. 主动出击寻找潜伏的优秀人员；
      3. 客观的评估；
      4. 给应聘者一个加入的理由；
  6. 授权于员工
    1. 消除地位象征；
    2. 依靠数据而不是经理的想法来做出决策；
    3. 探寻方法，让员工塑造自己的工作和公司；
    4. 高期待：决策由层级最下面的员工进行制定；（除非基于同样数据，上一层会做出不同的决定，才值得上报）
  7. 为什么每个人都讨厌绩效管理
    1. 关注个人成长，而不是评分和奖励，以此改善绩效；
    2. 今天多数绩效评价体系的最大问题，在于它经常替代了那些关注员工成长的重要活动（人们期望实践一个简单的办法，然后坐等美好的结果发生）；
    3. 正确的设定目标：通过OKR来统一和分解团队的目标；目标必须具体、可度量、可检验；如果你达成所有结果，就能完成目标；
    4. 收集同事的反馈意见（利用团队的力量，而且这样也更加能够确保公平）；（询问哪件事情可以做得更好，以及一件采取不同做法会产生更大影响的事情；一次只专注一件，效果更好）；
    5. 通过校准流程确保考评结果（利用几名经理人都熟悉的人，来设定考评基准）；（5-10名经理人一组，对下面的50-100名员工做考评；开会前提示一些常见错误及修正的办法）
    6. 将奖励分配谈话，与员工发展谈话分开；（避免由自发的成长，变成由外在的激励所左右，切记）
    7. 5级绩效考评：需要改进、达到期望、超出期望、大幅超出期望，表现杰出（我个人倾向只用4级，因为后两个的区别有点模糊，而且减少级数也有利于更多关注两端）；（如果不将期望写出来，感觉很多所谓的期望可能会变得很主观，最好是在期初的时候，就提前先将期望写好，而且最好让被评估人知道，这样双方不容易就努力的方向产生分歧）
    8. 给经理与员工的面谈提供指南，使对话更加具体和切合实际；
    9. 通过集体智慧来做出升职决定；对于升职不成功的自荐，给予反馈意见；
  8. 管理团队的两端：最优员工和最差员工
    1. 助力有难处的员工；
    2. 将最优秀的人放在显微镜下进行观察；（一个有趣的做法：获奖条件是分享经验）；
    3. 利用调查和检查清单寻找真相，推动员工学习；（通过设置合适的题目，向员工调查其经理的表现，以便获得信息进行针对性改进）；
    4. 与人分享员工对你的反馈意见，以身做则采取行动解决问题，身先示范（去参加一些针对性的培训）；
  9. 打造学习型组织
    1. 进行刻意的练习，将课程分成易于消化的小块，给出明晰的反馈意见，并不断重复这个过程；
    2. 请最优秀的员工教学；
    3. 只在已经证明能够改变员工行为的课程上投入；（可以采用对照组实验）
  10. 不公平薪酬
    1. 努力工作，但不炫耀（用球棒砸烂宝马车的例子）；
    2. 在公司小的时候（几百人），降低薪水，给予期权；在公司大的时候，再考虑采用有吸引力的薪水；
    3. 全员持股，奖励突出贡献的每一个人，而不是只局限于高管；
    4. 以成就为荣，不以报酬为荣（确保程序公正，比如引入同事评价；幂律分布，对突出贡献的人员加大奖励力度；公开进行体验奖励，私下进行现金和股权奖励）；
    5. 创造易于传播爱的环境（让给他人点赞变得更加便利，而且可以公开展示）；
    6. 精心筹划却遭遇失败的仍然要奖励（要鼓励敢于冒险的文化）；
  11. 世界上最好的东西是免费的
    1. 利用人力资源项目达成三个目标：效率、社区意识、创新精神；
    2. 提高职业和个人生活的效率：可能并不需要付出费用，只是让它们变得触手可及即可，让员工免去寻找它们的时间；
    3. 社区意识：让家人了解自己的工作、随机午餐、兴趣群组和俱乐部；（促进员工之间的交流，激发更多思想）
    4. 推动创新：创造促进员工交流的环境、开展各种演讲（促进思想的流动）；
    5. 想办法说“好”，凡事先尝试，再改进调整，而不是一开始就拒绝；
    6. 员工最需要你的时候请伸出援手：保险计划、延长产假；（给予员工生活上的关怀）
  12. 助推
    1. 利用助推帮助员工变得更明智：助人与求助的调查与排名（员工只知道自己的排名）；（利用社会比较力量：人之好胜与利他的本性）
    2. 帮助新员工融入集体，事项清单：
      1. 进行一次角色与责任的讨论；
      2. 将新员工与另一名同事组成互帮互助组；
      3. 帮助新员工建立社交网络；
      4. 前6个月每月进行一次上岗情况检查；
      5. 鼓励畅所欲言；
    3. 通过清单，减少需要记忆的事项的数量，可以让人对更重要的事情保持关注；
    4. 利用助推帮助员工变得更加富有：定期储蓄计划，如年金等；
    5. 利用助推帮助员工变得更加健康：餐盘大小的例子
    6. 建议做很多小实验，证明有效后再推广；
  13. 谷歌的教训
    1. 透明的代价：不可避免可能会造成机密信息的泄露；如果出现这种情况，毫不留情的进行惩罚；透明带来的好处，远超过它的代价；
    2. 摒弃应得的权利：如果出现了滥用的情况，公开的提出来，引起大家的注意力，让小恶接受公众的质疑和思考，从而不再容易发生；
    3. 失败的绩效改革：当出现危机的时候，停下手里所有的工作，处理危机；越是危机的时候，越是赢得他人信任与尊重的关键时候；
    4. 珍视怪人：有时他们会给有一些预见性和启发性的问题和思考；
    5. 有的放矢：平衡个人自由和公司整体方向的关键在于透明；同时还要解释更广阔的背景；
    6. 搞砸的时候：
      1. 承认错误，坦诚面对错误；
      2. 吸取各方面的意见；
      3. 不管什么坏掉了，修好；
      4. 找出错误中的寓意，加以传播；
